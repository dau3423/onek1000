## 2. 데이터베이스 스키마 (Supabase / PostgreSQL + PostGIS)

```sql
-- PostGIS 확장 활성화
create extension if not exists postgis;

-- 주유소 마스터 (정적/준정적 정보)
create table stations (
  id              text primary key,             -- Opinet UNI_ID
  brand_code      text not null,                -- SKE/GSC/HDO/SOL/RTE/ETC/RTO/E1G/SOG/NHO
  brand_name      text not null,                -- "SK에너지", "GS칼텍스" 등 표시용
  name            text not null,                -- 상호
  sido_code       text not null,                -- 01~19
  sigungu_code    text,
  address         text,
  zip             text,
  tel             text,
  is_self         boolean default false,        -- 셀프 여부
  has_carwash     boolean default false,
  has_cvs         boolean default false,        -- 편의점
  has_maintenance boolean default false,
  geom            geography(Point, 4326) not null,
  created_at      timestamptz default now(),
  updated_at      timestamptz default now()
);

create index stations_geom_idx on stations using gist (geom);
create index stations_sido_idx on stations (sido_code);
create index stations_brand_idx on stations (brand_code);

-- 가격 최신값 (자주 갱신되는 핫 테이블)
create table prices_latest (
  station_id text not null references stations(id) on delete cascade,
  product    text not null,                     -- B027/B034/D047/K015/C004
  price      integer not null,                  -- 원/L
  trade_dt   date not null,                     -- 가격 적용일
  updated_at timestamptz default now(),
  primary key (station_id, product)
);

create index prices_latest_product_price_idx on prices_latest (product, price);

-- 가격 이력 (선택, 베타 단계에서 추가)
create table prices_history (
  id         bigserial primary key,
  station_id text not null,
  product    text not null,
  price      integer not null,
  observed_at timestamptz default now()
);
create index prices_history_st_pd_at_idx
  on prices_history (station_id, product, observed_at desc);

-- 사용자 (NextAuth 호환)
create table users (
  id          uuid primary key default gen_random_uuid(),
  email       text unique,
  name        text,
  provider    text,                              -- google/kakao
  created_at  timestamptz default now()
);

-- 구독
create table subscriptions (
  id              uuid primary key default gen_random_uuid(),
  user_id         uuid not null references users(id) on delete cascade,
  status          text not null,                 -- trial/active/canceled/expired
  plan            text not null default 'monthly_1000',
  billing_key     text,                          -- 토스 자동결제 키 (암호화 저장)
  current_period_start timestamptz,
  current_period_end   timestamptz,
  canceled_at     timestamptz,
  created_at      timestamptz default now()
);
create index subscriptions_user_idx on subscriptions (user_id);
create index subscriptions_status_idx on subscriptions (status);

-- 즐겨찾기 (베타 단계)
create table favorites (
  user_id    uuid not null references users(id) on delete cascade,
  station_id text not null references stations(id) on delete cascade,
  created_at timestamptz default now(),
  primary key (user_id, station_id)
);
```

## 3. 핵심 쿼리

### 3.1 화면 영역(bbox) 내 최저가 TOP N
```sql
select s.id, s.name, s.brand_code, s.is_self,
       st_y(s.geom::geometry) as lat,
       st_x(s.geom::geometry) as lng,
       p.price, p.trade_dt
from stations s
join prices_latest p on p.station_id = s.id
where p.product = $1                                    -- 유종 코드
  and s.geom && st_makeenvelope($2, $3, $4, $5, 4326)  -- sw_lng,sw_lat,ne_lng,ne_lat
order by p.price asc
limit $6;                                                -- 줌별 N
```

### 3.2 반경 R 미터 내 최저가 TOP N
```sql
select s.id, s.name, s.brand_code,
       st_y(s.geom::geometry) as lat,
       st_x(s.geom::geometry) as lng,
       p.price,
       st_distance(s.geom, st_makepoint($1, $2)::geography) as distance_m
from stations s
join prices_latest p on p.station_id = s.id
where p.product = $3
  and st_dwithin(s.geom, st_makepoint($1, $2)::geography, $4)   -- $4=1000m
order by p.price asc, distance_m asc
limit $5;
```

### 3.3 시도별 평균가 (화면 zoom 1~6용)
- Opinet `avgSidoPrice.do`를 그대로 표시하는 캐시 응답으로 처리
- 별도 DB 쿼리 불필요. Redis에 `avg:sido:{prod}` 키로 캐시.

## 4. 캐싱 전략

### 4.1 캐시 레이어 구성
```
[Browser] ─ TanStack Query (memory, 5min) ─┐
                                            ├─ [Vercel Edge HTTP Cache] ─┐
                                            │                            ├─ [Upstash Redis] ─ [Supabase]
                                            └─ stale-while-revalidate ───┘
```

### 4.2 캐시 키 전략

| 키 패턴 | TTL | 무효화 |
|---------|-----|--------|
| `avg:nation` | 1h | Cron 동기화 종료 시 |
| `avg:sido:{prod}` | 1h | Cron 동기화 종료 시 |
| `avg:sigungu:{sidoCode}:{prod}` | 1h | Cron 동기화 종료 시 |
| `bbox:{geohash5}:z{zoom}:{prod}:n{N}` | 10min | TTL 만료에 위임 |
| `radius:{geohash7}:{prod}:r{R}` | 10min | TTL 만료에 위임 |
| `station:{id}` | 1h | UPDATE 시 명시적 삭제 |

**좌표 양자화 (geohash5 ≒ 5km 격자)**: 사용자가 지도를 미세하게 움직여도 같은 캐시 키 활용. 줌 12 이상에선 geohash6(1.2km) 사용.

### 4.3 데이터 신선도 vs 비용 트레이드오프

| 옵션 | 신선도 | 비용 | 선택 |
|------|--------|------|------|
| 매 요청마다 Opinet 호출 | 5분 | 매우 높음(API 한도 위험) | X |
| Cron 1h + Redis 캐시 10min | 최대 1h+10min | 낮음 | **선택** |
| Cron 15min + Redis 캐시 5min | 최대 20min | 중간 | 베타에서 검토 |

Opinet 자체가 1일 1~2회 보고이므로, 시간당 갱신이면 사실상 실시간에 가깝다.

### 4.4 Cron 동기화 로직 의사코드
```typescript
// POST /api/internal/sync-opinet
// 인증: Bearer ${CRON_SECRET}

async function syncOpinet() {
  const sidos = ['01', '02', /* ... */ '19'];
  const products = ['B027', 'D047']; // MVP는 휘발유/경유만

  // 1) 시도별 평균가
  for (const prod of products) {
    const avg = await opinet.avgSidoPrice({ prodcd: prod });
    await redis.setex(`avg:sido:${prod}`, 3600, JSON.stringify(avg));
  }

  // 2) 시도별 최저가 TOP20 → stations + prices_latest UPSERT
  for (const sido of sidos) {
    for (const prod of products) {
      const list = await opinet.lowTop10({ area: sido, prodcd: prod, cnt: 20 });
      for (const s of list) {
        await db.upsertStation(s);     // id 변경 없으면 noop
        await db.upsertPriceLatest({
          station_id: s.UNI_ID,
          product: prod,
          price: s.PRICE,
          trade_dt: s.TRADE_DT,
        });
      }
    }
  }

  // 3) 캐시 invalidate (필요한 키만)
  await redis.del('avg:nation');
  // bbox/radius는 TTL 만료에 위임
}
```

### 4.5 초기 시드 (1회성)
MVP 출시 직전에 전국 주유소 전체 마스터 데이터를 채워둬야 한다. 무료 API는 "전국 주유소 마스터 전체 다운로드"가 없고 시도별 TOP20만 제공하므로 다음 중 하나 선택:

1. **공공데이터포털 CSV 다운로드 활용** — "한국석유공사_전국 주유소 평균가격" 또는 일부 데이터셋에서 CSV로 마스터 수급 (가능성 검토 필요)
2. **유료 API 신청 (주유소 기본정보 지역별)** — 정식 운영 시 추천
3. **MVP에서는 TOP20만 운영** — "내 화면에 보이는 최저가만 보이면 된다"는 컨셉상 운영 가능. 단, 화면에 보이는 주유소가 듬성듬성할 수 있음.

→ **MVP 권장**: 옵션 3으로 시작 + Opinet `aroundAll.do`(반경 검색)로 지도 영역 내 보충. 베타 진입 시 옵션 2로 전환.

## 5. 응답 데이터 포맷 (API → 클라이언트)

```typescript

type BboxResponse = {
  stations: Station[];
```

## 6. 오피넷 데이터 한계와 대응

| 한계 | 대응 |
|------|------|
| 시도/시군구별 TOP20만 무료 제공 | MVP는 TOP20 합집합으로 운영, 베타에 유료 API |
| XML 응답이 기본 | `out=json`으로 받되, XML 폴백 파서 준비 |
| 좌표가 KATEC(TM)일 수 있음 | 응답 좌표계 확인 후 WGS84 변환 (proj4js) |
| 호출 한도 명시 없음 | 시간당 100회 이내로 자체 제한 (50회 × 안전마진) |
| 일부 가격 정보 지연 | UI에 `trade_dt` 노출하여 신선도 명시 |
