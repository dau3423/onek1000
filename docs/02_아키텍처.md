# 1000냥 주유소 - 시스템 아키텍처

## 1. 기술 스택 요약

| 레이어 | 기술 | 선정 이유 |
|--------|------|----------|
| 프레임워크 | Next.js 14 (App Router) | SSR/ISR로 초기 로딩 최적화, API Routes로 백엔드 통합, Vercel 호스팅과 궁합 |
| 언어 | TypeScript 5 | 타입 안정성, 13년차 개발자 작업 효율 |
| UI | React 18 + Tailwind CSS | 빠른 프로토타이핑, 모바일 우선 디자인 |
| 상태관리 | Zustand + TanStack Query | 가벼움, 서버 상태 캐싱 최적화 |
| 지도 | 카카오맵 JS SDK | 한국 주유소 좌표 정확도, MarkerClusterer 내장, 무료 한도 충분 |
| DB / 캐시 | Supabase (PostgreSQL + PostGIS) + Upstash Redis | PostGIS로 공간쿼리, Redis로 핫데이터 캐싱 |
| 배포 | Vercel (Web) + Supabase (DB) + Upstash (Redis) | 모두 무료 티어로 MVP 가능 |
| 결제 | 토스페이먼츠 빌링(정기결제) | 국내 정기결제 안정성, 카드 등록 후 자동결제 |
| 광고 | Google AdSense | 모바일웹 매칭 단가 양호, 통합 간단 |
| 모니터링 | Sentry + Vercel Analytics | 에러 추적 + 사용자 행동 |

## 3. 데이터 플로우

### 3.1 일반 사용자 지도 패닝 시
1. 사용자가 지도 패닝/줌 → bounds(SW/NE 좌표) + 줌 레벨 + 유종 결정
2. 클라이언트 → `GET /api/stations/bbox?sw=...&ne=...&zoom=12&prod=B027&limit=100`
3. API Route → Redis 캐시 키(`bbox:{좌표양자화}:zoom:{z}:prod:{p}`) 조회
4. Hit → 즉시 응답 / Miss → Supabase PostGIS `ST_Within` 쿼리 → 결과 캐시 후 응답
5. 클라이언트가 받은 JSON 배열을 카카오맵 마커로 렌더, MarkerClusterer로 묶음

### 3.2 GPS 1km 알람
1. 사용자 위치 동의 → `navigator.geolocation.watchPosition`
2. 좌표가 의미있게 변하면(100m 이상 이동) → `GET /api/stations/radius?lat=...&lng=...&r=1000&prod=B027`
3. API Route → Redis 캐시 (좌표 100m 양자화) → Miss 시 PostGIS `ST_DWithin` 쿼리
4. 결과 중 가격이 화면 영역 평균보다 50원 이상 저렴한 항목이 있으면 → 클라이언트가 상단 배너 알람

### 3.3 Cron으로 데이터 동기화
1. Vercel Cron 1시간 주기 (`0 * * * *`) → `/api/internal/sync-opinet` 호출
2. 동기화 로직:
   - 시/도 17개 × 유종 2~3개(B027 휘발유, D047 경유) = ~50회 API 호출
   - 각 시도별 평균가 + 시도별 최저가 TOP 20 가져옴
   - 변경된 주유소만 `prices_latest` UPSERT
3. 동기화 종료 시 Redis 캐시 invalidate (선택적 — TTL 만료에 맡겨도 됨)

**참고**: 오피넷 무료 API는 호출량 명시가 없으나, 시간당 수백 회 수준은 안전. 50회/시간은 충분히 여유.

## 6. 보안 고려사항

- Opinet API 키는 서버 사이드에서만 사용 (`OPINET_API_KEY`, public 아님)
- `/api/internal/*`는 `Authorization: Bearer ${CRON_SECRET}` 검증
- 광고 SDK는 무료 사용자에게만 로드 (구독자 정보는 서버 검증)
- 카카오맵 JS 키는 도메인 허용 목록을 카카오 디벨로퍼스에서 잠금
- Rate Limit: `/api/stations/*` IP당 분당 60회 (Upstash Ratelimit 활용)

## 7. 성능 전략

| 병목 | 대응 |
|------|------|
| Opinet API 매번 호출 | Cron으로 1h 주기 사전 동기화 → DB에서 서빙 |
| PostGIS 쿼리 비용 | `geom`에 GiST 인덱스, 자주 쓰는 bbox 결과 Redis 캐시 |
| 마커 100개 렌더 부담 | 카카오 MarkerClusterer로 줌 낮을 때 묶음, CustomOverlay는 줌 12+에서만 |
| 가격 라벨 SVG 비용 | 줌별 노출 개수 제한 + Canvas/CSS 텍스트로 경량화 |
| 초기 JS 번들 | 카카오맵 SDK는 dynamic import + loading="lazy", 지도 페이지에서만 |
| 광고 SDK 영향 | AdSense는 `next/script` strategy="lazyOnload" |
