# 1000냥 주유소 - 단계별 개발 계획

## 0. 사전 준비 (Day 0)

| 항목 | 주체 | 비고 |
|------|------|------|
| 도메인 등록 (`onek1000.kr`) | 운영 | 가비아/Cloudflare |
| 카카오 디벨로퍼스 앱 등록 + JS 키 발급 | 개발 | 도메인 추가 등록 |
| 오피넷 무료 API 이용신청 + 인증키 발급 | 운영 | 1~3영업일 소요 |

## 마일스톤 1: MVP (1~2주)

**목표**: 지도 위에 가격 마커 + GPS 1km 알람이 동작하는 단일 페이지

### Week 1 - 기반 + 지도
- [ ] Next.js 14 프로젝트 부트스트랩 (App Router, TS, Tailwind, ESLint)
- [ ] 환경변수 템플릿 / `.env.local` 작성
- [ ] Supabase 프로젝트 생성, PostGIS 확장, `stations`/`prices_latest` 마이그레이션 적용
- [ ] Upstash Redis 연결 클라이언트
- [ ] `lib/opinet/client.ts`: Opinet 무료 API 클라이언트 (xml/json 파서, 시도/유종 코드 상수)
- [ ] `scripts/seed-stations.ts`: 17개 시도 × 휘발유/경유 TOP20 → 약 680개 주유소 시드
- [ ] `components/map/KakaoMap.tsx`: 지도 컨테이너 (dynamic import, SSR off)
- [ ] `app/page.tsx`: 메인 지도 화면 SSR 골격 + 기본 마커

### Week 2 - API + 상호작용
- [ ] `GET /api/stations/bbox`: PostGIS bbox 쿼리 + Redis 캐시
- [ ] `GET /api/stations/radius`: PostGIS dwithin 쿼리
- [ ] 카카오 MarkerClusterer 적용 (줌 1~9 클러스터, 10+ 개별)
- [ ] 가격 라벨 CustomOverlay (줌 12+)
- [ ] 유종 필터 바 (Zustand store)
- [ ] 내 위치 버튼 + watchPosition 훅
- [ ] 1km 알람 배너 (조건: 평균 -50원 이상)
- [ ] `GET /api/stations/[id]` + 상세 페이지(미니맵, 가격, 길찾기 외부링크)
- [ ] Vercel Cron 등록 (`POST /api/internal/sync-opinet` 매시간)
- [ ] 빌드 + Vercel 배포 + 도메인 연결

**완료 기준**: 모바일 크롬에서 위치 동의 후 지도에 주변 주유소 가격이 보이고, 더 싼 곳을 발견하면 상단 배너로 알려준다.

## 마일스톤 2: 알파 (3~4주차)

**목표**: 로그인/구독/광고가 실제 매출 발생 가능한 형태로 동작

### 인증 / 결제
- [ ] NextAuth.js + 카카오 / 구글 OAuth 연동
- [ ] `users` 테이블 동기화 (Supabase Adapter)
- [ ] `subscriptions` 테이블 + 토스페이먼츠 빌링키 발급 플로우 (`POST /api/billing/subscribe`)
- [ ] 7일 무료 체험 로직 + 자동결제 스케줄러 (Vercel Cron 일 1회)
- [ ] 결제 실패/재시도/취소 처리
- [ ] `POST /api/billing/webhook`: 토스 결제 결과 수신 → DB 업데이트
- [ ] 마이페이지 `/my`: 구독 상태 표시, 해지 버튼

### 광고
- [ ] AdSense 계정 발급 후 클라이언트 ID 연결
- [ ] `components/ads/BannerAd.tsx`: 지도 하단 sticky 배너 (무료 사용자만)
- [ ] `components/ads/InterstitialAd.tsx`: 상세 진입 시 1일 1회 (localStorage flag)
- [ ] 구독자 판정 SSR 헬퍼 (`isPremium(session)`) → 광고 컴포넌트 분기
- [ ] 광고 SDK 로딩 실패 fallback (페이지 영향 0)

### UX 보강
- [ ] 하단 시트 컴포넌트 (TOP N 리스트, 드래그)
- [ ] 다크모드 토큰
- [ ] 로딩 스켈레톤
- [ ] 에러 페이지 (`/error`, `not-found`)
- [ ] Sentry 연동 + 에러 알림 설정

### 운영 준비
- [ ] 약관 / 개인정보처리방침 (변호사 검토 또는 표준 템플릿 + 위치/결제 항목 명시)
- [ ] 오피넷 출처 표기 + 데이터 마지막 갱신 시각 노출
- [ ] 카카오/구글 OAuth 콘솔 도메인 등록
- [ ] 토스페이먼츠 실사용자 키 발급 (사업자등록 필요)

**완료 기준**: 사용자가 회원가입→7일 체험→자동결제까지 흐를 수 있고, 무료/유료 분기가 광고 노출에 정확히 반영된다.

## 마일스톤 3: 베타 (5~8주차)

**목표**: 사용자 데이터/이력 기반 부가 기능과 운영 안정성

- [ ] 즐겨찾기 (`favorites` 테이블, 마이페이지 동기화)
- [ ] 가격 이력 (`prices_history` 적재 + 상세 페이지 7일 차트)
- [ ] 웹 푸시 알림 (FCM, 즐겨찾기 가격 변동) — 유료 한정
- [ ] 검색 기능: 주유소 상호/주소 검색 (`/api/search`)
- [ ] 경로별 최저가 (출발/도착 입력 → 경로 위 주유소 정렬)
- [ ] PostGIS 인덱스 튜닝, 슬로우 쿼리 모니터링
- [ ] Opinet 유료 API "주유소 기본정보(지역별)"로 전환 → 전국 전체 주유소 마스터 확보
- [ ] 데이터 신선도 표시 강화 (각 주유소 카드에 보고일자)
- [ ] 부하 테스트 (k6 또는 artillery, 동시 1만 가상 사용자)
- [ ] WCAG AA 접근성 점검

## 위험 요소 및 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|
| Opinet 유료 API 협의 지연 | 전국 마스터 데이터 부족 → 일부 주유소 누락 | MVP는 TOP20 합집합 + 반경검색으로 운영, 협의 병행 |
| 오피넷 상업적 이용 거절 | 사업 중단 위험 | 출시 전 사전 협의 필수, 거절 시 직접 크롤링/제휴 주유소 데이터 검토 |
| 카카오맵 트래픽 초과 | 지도 미표시 | 카카오 비즈 전환 (유료) — DAU 10만 이전엔 무료 한도 내 |
| 광고 SDK가 페이지 깨뜨림 | 사용자 이탈 | 광고 컴포넌트 ErrorBoundary, 실패 시 빈 div |
| 정기결제 실패 누적 | 매출 손실 | 토스 웹훅 + 재시도 큐 + 사용자 알림 메일 |
| 위치 권한 미동의 | 핵심 가치 훼손 | 디폴트 도시(서울 시청) + 수동 검색으로 우회 |
| 가격 정보 오차로 사용자 신뢰 하락 | 이탈 + 컴플레인 | 각 카드에 "마지막 보고일" 표기, 컴플레인 채널 운영 |

## 작업 우선순위 — 첫 주 권장 순서

1. **Day 1-2**: Next.js 부트스트랩 + 카카오맵 표시 + 더미 마커 5개
2. **Day 3-4**: Opinet 클라이언트 + 시드 스크립트 + Supabase 적재
3. **Day 5-6**: `/api/stations/bbox` + 지도 패닝 시 마커 갱신 + Redis 캐시
4. **Day 7-8**: 1km 알람 + 유종 필터 + 상세 페이지
5. **Day 9-10**: Vercel 배포 + 도메인 + Cron + 폴리싱

MVP까지 풀타임 1명 기준 약 10영업일.
